### 1. 前端功能修复

#### 1.1 Ctrl+ 单击多选功能修复
**问题**：Ctrl+ 单击无法正确累加选中报价，单击会清空所有选中项

**修复内容**：
- 重构 `toggleSelect` 函数，正确处理 `useCtrl` 参数
- 单击：清空之前选中，只选中当前项
- Ctrl+ 单击：累加选中（类似文件管理器行为）

**相关代码** (`frontend/App.tsx:834`)：
```javascript
const toggleSelect = (id: string, isChecked?: boolean, useCtrl = false): void => {
  setSelectedQuotes(prev => {
    const newSet = new Set(prev);
    const targetState = isChecked !== undefined ? isChecked : !newSet.has(id);
    if (useCtrl) {
      if (targetState) {
        newSet.add(id);
      } else {
        newSet.delete(id);
      }
    } else {
      newSet.clear();
      if (targetState) {
        newSet.add(id);
      }
    }
    return newSet;
  });
};
```

#### 1.2 拖曳选择后自动复制功能
**问题**：拖曳选中报价后，需要手动复制，操作繁琐

**修复内容**：
- 添加 `userChangedSelection` 状态标志
- 使用 `useEffect` 在选中状态稳定后自动触发复制
- 解决 React 异步状态更新导致的竞态问题

**相关代码** (`frontend/App.tsx`):
```javascript
const [userChangedSelection, setUserChangedSelection] = useState(false);

useEffect(() => {
  if (userChangedSelection && selectedQuotes.size > 0) {
    copySelectedQuotes(selectedQuotes);
    setUserChangedSelection(false);
  }
}, [selectedQuotes]);

// onClick  handlers 中设置标志而非直接调用 copy
onClick={(e) => {
  e.stopPropagation();
  const isCtrl = e.ctrlKey;
  setUserChangedSelection(true);
  toggleSelect(item.id, undefined, isCtrl);
}}
```

#### 1.4 收益率精度显示修复
**问题**：收益率显示 1.560 应该显示为 1.56（第三位是 0 时省略）

**修复内容**：
- 修改 `parser.ts` 中的 `findYieldRate` 函数
- 规则：最小 2 位小数，最多 4 位，第三位是 0 时保留 2 位

**相关代码** (`frontend/services/parser.ts`):
```javascript
let formattedRate = rateVal.toFixed(4);
formattedRate = formattedRate.replace(/(\.\d{2,})0+$/, '$1');
const decimalPart = formattedRate.split('.')[1];
if (decimalPart && decimalPart.length === 1) {
  formattedRate = rateVal.toFixed(2);
}
// 特殊处理：如果第三位是 0，只保留 2 位（1.560 → 1.56）
if (decimalPart && decimalPart.length >= 3) {
  const thirdDigit = decimalPart[2];
  if (thirdDigit === '0') {
    formattedRate = rateVal.toFixed(2);
  }
}
```

### 2. 后端安全配置

#### 2.1 后端监听地址修改
**问题**：后端监听 `0.0.0.0:3000`，直接暴露到公网，存在安全风险

**修复内容**：
- 修改 `backend/.env`，添加 `HOST=127.0.0.1`
- 修改 `backend/src/index.js`，使用 HOST 参数绑定监听地址
- 重启后后端只监听本地回环地址，不直接暴露

**修改前**：
```javascript
let PORT = process.env.PORT || 3000;
httpServer.listen(PORT, () => { ... });
```

**修改后**：
```javascript
let PORT = process.env.PORT || 3000;
let HOST = process.env.HOST || '127.0.0.1';
httpServer.listen(PORT, HOST, () => { ... });
```

**验证命令**：
```bash
netstat -tlnp | grep 3000
# 正确输出：tcp  0  0  127.0.0.1:3000  0.0.0.0:*  LISTEN  12345/node
```

## 技术细节

### React 状态更新竞态问题
**问题**：在 `onClick` 中直接调用 `copySelectedQuotes` 会读取到旧的 state

**原因**：React 的 `setState` 是异步的，状态更新需要时间

**解决方案**：
1. 使用标志位 `userChangedSelection` 记录用户操作
2. 使用 `useEffect` 在状态稳定后执行复制

---

### 双击编辑实现

1. `onDoubleClick` 中调用 `e.stopPropagation()` 防止事件冒泡
2. 设置 `editingId` 进入编辑模式
3. 使用 `(e.target as HTMLInputElement).focus()` 自动聚焦
4. `onBlur` 时退出编辑模式
